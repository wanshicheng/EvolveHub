# EvolveHub 桌面宠物系统设计文档

## 1. 概述

EvolveHub 桌面宠物是一个纯前端实现的 AI 伴侣角色，为普通用户的"AI Space"桌面注入生命力。每个用户拥有独一无二、自动生成的宠物伙伴，具备行走、睡眠、情绪表达和交互能力。管理员可通过专用面板查看所有用户的宠物。

### 设计理念

- **差异化** — 业界首创的 AI 中台桌面宠物，打造产品辨识度
- **情感连接** — 从"使用工具"变为"养一个伙伴"，提升用户留存
- **轻量实现** — 纯 CSS + Vue 响应式，零外部依赖，不引入游戏引擎
- **渐进增强** — 分阶段构建，当前为第二阶段（物理行走 + 状态机）

### 3.1 自动生成规则

宠物外观由用户 ID 取模决定，确保同一用户每次看到相同的宠物：

```typescript
颜色 = colorPalettes[userId % 8]
配件 = accessories[userId % 5]
名字 = petNames[userId % 16]
等级 = 3 + (userId % 8)
```

## 4. 状态机系统

### 4.1 状态定义

```
         ┌──────────────────────────────────┐
         │                                  │
         ▼                                  │
      ┌──────┐  40%概率   ┌──────────┐     │
      │ idle │ ────────▶ │ walking  │     │
      │(待机)│ ◀──────── │ (行走中) │     │
      └──┬───┘  3-6秒后  └──────────┘     │
         │                                  │
         │ 45秒无操作                        │
         ▼                                  │
      ┌──────────┐   点击/用户回来          │
      │ sleeping │ ──────────────────┐      │
      │ (睡眠中)  │                   │      │
      └──────────┘                   ▼      │
         ▲                    ┌──────────┐  │
         │                    │ greeting │  │
         │                    │ (欢迎中)  │  │
         │                    └────┬─────┘  │
         │                         │3秒后    │
         │                         ▼        │
         │                      ┌──────┐    │
         └──────────────────────│ idle │────┘
                                └──┬───┘
                                   │对话窗口打开
                                   ▼
                                ┌──────────┐
                                │ working  │
                                │ (陪伴中)  │
                                └────┬─────┘
                                     │窗口关闭
                                     ▼
                                  ┌──────┐
                                  │ idle │
                                  └──────┘
```

### 4.2 状态详细说明

#### idle（待机）

- **触发**：默认状态 / 其他状态结束后
- **行为**：站着不动，浮动动画
- **决策**：每 3-8 秒随机决定下一步
  - 40% → walking（走起来）
  - 15% → curious 好奇（"嗯？"）
  - 15% → blush 害羞（红晕）
  - 30% → 继续 idle

#### walking（行走）

- **触发**：从 idle 随机决策（40%概率）
- **行为**：在屏幕底部左右走动，身体摇晃步行动画
- **方向**：自动面向行走方向（`scaleX(-1)` 翻转）
- **时长**：3-6 秒后停下，回到 idle
- **移动**：CSS `transition: left 2.5s linear` 平滑位移

#### sleeping（睡眠）

- **触发**：45 秒无用户操作
- **行为**：
  - 闭眼（眼睛变为横线）
  - 缓慢呼吸动画（`scale(1.05)`）
  - zZZ 浮动文字（三个 z 交错上升）
  - 阴影变大变淡
- **台词**："困了...zzZ"
- **唤醒**：点击宠物 / 任何鼠标/键盘操作

#### greeting（欢迎）

- **触发**：从睡眠中唤醒 / 用户离开后回来
- **行为**：
  - 高兴蹦跳动画
  - 挥手
  - 随机欢迎台词
- **台词**："你回来啦！"、"想你了~"、"好久不见！"、"呜呜呜好想你~"
- **时长**：3 秒后回到 idle

#### working（陪伴）

- **触发**：用户打开对话窗口
- **行为**：
  - 走到屏幕右下角安静坐着
  - 微弱脉冲动画
  - 每 8-15 秒冒一句鼓励语
- **台词**："加油哦！"、"需要帮忙随时叫我~"、"你做得很好！"、"默默陪伴中..."
- **结束**：对话窗口关闭后回到 idle

## 7. 交互设计

### 7.1 用户交互

| 操作 | 宠物反应 |
|------|---------|
| 鼠标移动 | 眼睛追踪鼠标（限制 ±4px 水平 / ±3px 垂直） |
| 点击宠物 | 弹跳 + 挥手 + 随机台词 |
| 睡眠中点击 | 唤醒 + 欢迎动画 |
| 45 秒不操作 | 宠物入睡 |
| 操作恢复 | 宠物跳起欢迎 |
| 打开对话窗口 | 宠物走到角落安静陪伴 |
| 关闭对话窗口 | 宠物回到 idle |

### 7.2 空闲检测

追踪以下事件以重置空闲计时器：
- `mousemove` — 鼠标移动
- `click` — 鼠标点击
- `keydown` — 键盘按键

空闲阈值：**45 秒**（之后进入 sleeping 状态）

## 9. 技术实现细节

### 9.1 位置系统

宠物在屏幕底部行走，使用 CSS `transition` 实现平滑移动：

```css
transition: left 2.5s linear;  /* 行走时 */
transition: left 0.5s ease;    /* 其他状态 */
```

行走目标随机生成，避开屏幕边缘（100px 边距）。

### 9.2 方向翻转

通过 `scaleX(-1)` 实现左右翻转，无需维护两套朝向：

```css
.pet-creature.face-left {
  transform: scaleX(-1);
}
```

### 9.3 窗口感知

宠物通过 `useWindowStore` 监听对话窗口状态：

```typescript
const isChatOpen = computed(() =>
  Object.values(winStore.windows).some(w => w.appId === 'chat' && w.isOpen)
)
```

### 9.4 无障碍

- `pointer-events: none` 在舞台层，`auto` 仅在宠物本身
- `user-select: none` 防止误选文字
- z-index: 12 确保在侧边栏之上但在窗口之下
